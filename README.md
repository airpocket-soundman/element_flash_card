# element_flash_card

元素を覚えるためのシンプルなフラッシュカード Web アプリです。
GitHub Pages でそのまま公開できる静的サイト構成です。

## 公開ページ

https://airpocket-soundman.github.io/element_flash_card/

ブラウザでアクセスするとそのまま使えます。PWA 対応のため、ホーム画面に追加すればアプリのように起動でき、オフラインでも動作します。

## 使用技術 / サービス

- **HTML / CSS / JavaScript**: フレームワーク非依存の素の Web 技術のみで構成
- **PWA**: `manifest.webmanifest` + `service-worker.js` によるインストール対応・オフライン動作
- **localStorage**: 学習履歴のローカル保存
- **GitHub Pages**: `main` ブランチの `/docs` フォルダを公開ソースとして配信
- ビルドツール / 外部依存なし。`docs/` をそのまま配信するだけで動作します。

## 機能

- 4 つの学習コース
  - 元素番号コース
  - 元素名(英語)コース
  - 元素名(日本語)コース
  - 元素記号コース
- 出題範囲を元素番号で指定可能（例: 1〜40）
- 1 問ごとに、選んだコースの項目だけ先に表示
- `答えを見る` で残り 3 項目を表示
- 自己採点で `正解` / `誤答` を記録して次の問題へ
- リザルト画面で各コースのワースト 10（誤答率が高い順）を表示
- 履歴は各コースごとにローカル保存（過去 10000 問を保持）
- 表示集計は `10000 / 5000 / 1000 / 500` 問で切替可能
- すべての画面に `メニューに戻る` ボタンを配置

## 学習ルール

- 出題はコース選択時に指定した元素番号の範囲からランダムに 1 問ずつ。
- 出題項目は選んだコースに対応する 1 項目のみが先に表示されます。
- `答えを見る` を押すと残りの 3 項目が表示されます。
- 正誤は **自己採点方式**。`正解` / `誤答` ボタンで自分で記録します。
- 採点結果はコース別に履歴として蓄積され、誤答率の高い元素がリザルトに表示されます。

## ファイル構成

- `docs/index.html`: 画面構成（メニュー / クイズ / リザルト）
- `docs/styles.css`: スタイル
- `docs/app.js`: 出題ロジック、履歴保存、リザルト集計
- `docs/elements.js`: 元素データ（番号 / 英語名 / 日本語名 / 記号）
- `docs/manifest.webmanifest`: PWA マニフェスト
- `docs/service-worker.js`: オフラインキャッシュ用 Service Worker
- `docs/icon.svg`: アプリアイコン

## 使い方（ローカル）

1. このリポジトリをダウンロードまたはクローン
2. `docs/index.html` をブラウザで開く
3. メニューで出題範囲を指定し、コースを選択して開始
4. `答えを見る` → `正解` または `誤答` を押して学習
5. `リザルトを見る` で弱点元素を確認

## GitHub Pages で配信する

1. GitHub にこのリポジトリを push
2. GitHub の `Settings` → `Pages`
3. `Build and deployment` の `Source` を `Deploy from a branch` に設定
4. Branch を `main`、フォルダを `/docs` に設定して保存
5. 数分後に公開 URL（ https://airpocket-soundman.github.io/element_flash_card/ ）へアクセス

## データ保存仕様

- 保存先: ブラウザの `localStorage`
- キー: `element_flash_card_history_v1`
- 保存単位: コース別履歴
- 1 コースの保持上限: 10000 件（超えた分は古い履歴から削除）

## 注意

- 正誤判定は自己採点方式です。
- `localStorage` を削除すると履歴も消えます。
- PWA のキャッシュ更新は Service Worker の挙動に依存します。最新版が反映されない場合はブラウザのキャッシュをクリアしてください。
